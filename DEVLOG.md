- Frontend: Vue 3 + Vuetify 3 + TypeScript + Vue Router, built with Vite
- Docker: Multi-stage Dockerfile (Node build + Python runtime), docker-compose.yml

**Phase 2 — Encryption Service:**

- 5 tests: round-trip, uniqueness, empty string, wrong key, missing key

**Phase 3 — Config Validation and App Startup:**
- 5 stub routers (auth, calendars, sync_pairs, sync, icloud) wired into the app
- All endpoints respond and appear in Swagger UI at `/docs`

### Decisions made beyond CLAUDE.md

1. **`pydantic-settings`** was added as a dependency — required by the Pydantic `BaseSettings` class which moved to its own package in Pydantic v2.

2. **Alembic migration template** (`script.py.mako`) was updated to include `import sqlmodel` since autogenerated migrations reference `sqlmodel.sql.sqltypes.AutoString`.

3. **`GAPPLE_STATIC_DIR` env var** is checked in `main.py` for Docker deployment, falling back to the development-time `frontend/dist/` path. This allows the Dockerfile to place built static files at `/app/static`.

4. **Python 3.11 compatibility** — the environment runs Python 3.11 rather than 3.12+. The code uses `X | None` union syntax which requires `from __future__ import annotations` on 3.9 but works natively on 3.10+. No issues encountered.

5. **Docker build** could not be verified in this environment (no Docker daemon). The Dockerfile follows the spec and should work.
